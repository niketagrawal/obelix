run_crest(path_to_complexes=[], path_to_output=[], conformer_search='off')[source]

This function is designed to either run conformer search with xtb preopt. or just xtb optimization. The subprocess.call is used to make a binary call from your ./bashrc file or any local path in your $PATH system variable.

Files are stored in the users preferred place when you run the workflow in a modular way, or by design the results are stored in your Workflow/CREST folder.

calculate_descriptors()[source]

This function uses the Descriptor class to calculate descriptors for xyz, CREST or log files. The output_type is used to determine which descriptors are calculated. The extract_xyz_from_log is used to extract the xyz coordinates from the log file. The printout is used to print the descriptors to the screen.

By default, self.path_to_workflow is used as the path for the files to be read in. (so obelix/Workflow)

Parameters:

  • metal_adduct – pristine, nbd are fully supported, acetonitrile is WIP

  • output_type – xyz, crest or gaussian

  • extract_xyz_from_log – True or False (only used for descriptors from gaussian output)

  • printout – True or False (prints the descriptors to the screen)

Returns:

dataframe of descriptors, calculated and loaded in the Descriptor class
